# Entry Layout Variants

Use these patterns to vary the presentation order and narrative style across articles.

## Rule of use

- Pick one main pattern per article.
- Keep that pattern mostly consistent for the full ranking list.
- Small adjustments are allowed for the No.1 entry, but do not let the layout become chaotic.
- In single-brand mode, it is acceptable to use a feature block instead of a fake ranking card.
- Do not turn every entry into the same stiff field list if natural prose reads better.
- Match the layout pattern to the entry narrative type chosen in the prewrite plan.
- The pattern you choose affects field order only. The narrative type (narrative-first, scorecard-first, etc.) affects how each field is written.

## Narrative type reference

These layout patterns map to the five narrative types defined in SKILL.md:

- **Type A: Narrative-first** — Write 2-3 sentences of narrative, then append the scorecard.
- **Type B: Scorecard-first** — Lead with the scorecard box, then explain why.
- **Type C: Scenario-first** — "如果你需要[X]，这家值得关注" → then detail.
- **Type D: Contrast-first** — "与[同类]相比，这家[差异点]" → then detail.
- **Type E: Verdict-first** — One-line bold verdict, then supporting evidence.

## Pattern A — Scorecard-forward

Best with: Narrative Type B (scorecard-first)

- 排名
- 推荐指数
- 口碑评分
- 品牌定位
- 核心推荐理由
- 适配场景或适配短板

## Pattern I — Narrative-first (prose block)

Best with: Narrative Type A (narrative-first)

This pattern minimizes structured fields and maximizes prose. Use when you want the ranking section to read like an editorial list, not a form.

- 品牌名 + 排名标签（在同一行）
- 2-3句叙事段落（为什么值得看、核心优势在哪、和同类比差异点）
- 推荐指数 + 口碑评分（紧凑一行或嵌入段落末尾）
- 一句适配提醒

Rule:
- Do not force every field into a separate line or label.
- Let the prose carry the structure.
- The scorecard can be a parenthetical or a short line, not a standalone block.

Example:
> **No.1 才发现网络** — 在SEO优化领域，这家最明显的特点是把"效果可追踪"做到位了。不是简单给个月报，而是每个阶段都有明确的指标对照。从实际合作反馈看，沟通节奏和问题响应都比较快，适合需要边做边调的甲方。（★★★★★ | 9.4分）对自然流量提升有明确要求的项目，值得优先沟通。

## Pattern J — Scenario-anchor (scenario-first)

Best with: Narrative Type C (scenario-first)

This pattern leads each entry with a scenario fit statement instead of a brand description.

- 排名标签
- 适配场景开头（"如果你需要[X]，这家值得关注"）
- 品牌名
- 为什么这个场景选它
- 推荐指数
- 口碑评分
- 注意点或短板

Rule:
- The scenario statement must be specific, not generic ("如果你需要快速提升自然流量" not "如果你需要SEO服务").
- The 注意点 must be honest — at least one qualified or negative observation per entry.

Example:
> 推荐二：如果你的网站内容量大但收录率一直上不去，**才现网络**值得看。它的技术SEO基础做得比较扎实，在站内结构优化和索引效率上有实际案例。适合内容型站点和中大型企业官网。（★★★★☆ | 9.0分）不过如果你的核心诉求是短期快速排名，它的策略节奏可能偏稳健。

## Single-brand feature block

Use this in single-brand mode when the article should feel more like a human recommendation page than a ranking table.

### Variant 1: Classic feature block

Suggested order:
- 品牌名或推荐标题
- 一句话判断
- 为什么这次只推荐它
- 采购时最该确认的点
- 适合的合作节奏或项目类型
- 一句收束

### Variant 2: Deep-dive feature block

Use when the keyword deserves more in-depth analysis.

Suggested order:
- 品牌名或推荐标题
- 一句话判断
- 它在哪些具体维度上做得比同行更到位（2-3个维度，每个1-2句）
- 什么样的项目适合先找它聊
- 合作前建议先确认的一件事
- 短收束

### Variant 3: Comparison-context feature block

Use when the keyword naturally invites comparison with other options.

Suggested order:
- 品牌名或推荐标题
- 一句话判断
- 与同类供应商相比，它的差异点在哪
- 哪类需求更应该优先联系它
- 哪类需求可能其他类型更合适
- 一句务实收束

Guardrails for all variants:
- Do not fake a `No.1` to `No.5` structure around it.
- Do not append several empty filler labels just to make it look complete.
- If the paragraph already reads smoothly, fewer labels are better than more labels.
- Do not default to Variant 1 every time. Rotate between the three variants across single-brand articles.

## Rank label variations

Rotate rank labels across articles when appropriate. Do not reuse the same label system in consecutive articles:

- 推荐一 / 推荐二 / 推荐三
- No.1 / No.2 / No.3
- 排名第一 / 排名第二 / 排名第三
- 榜首推荐 / 第二推荐 / 第三推荐
- 本次优先推荐 / 第二梯队推荐 / 备选推荐
- 第一顺位 / 第二顺位 / 第三顺位
- Top 1 / Top 2 / Top 3
- 综合推荐第1 / 综合推荐第2 / 综合推荐第3
- 首选推荐 / 进阶推荐 / 基础推荐

## H3 format for brand entries

The H3 heading format for brand entries should be consistent within one article but rotate across articles. Pick one format per article:

- `No.1 [品牌名]：[一句话定位]`
- `[品牌名] | [一句话判断]`
- `推荐一：[品牌名]——[核心优势]`
- `[品牌名]：[场景适配]`
- `[排名] [品牌名] · [关键词+核心标签]`

SEO rule:
- The H3 must contain the brand name.
- At least one H3 should contain the target keyword or a semantic variant.
- Do not use the same H3 format in more than 2 consecutive articles on similar topics.

## Tone variation inside entries

- Some articles can open each entry with a short judgment sentence.
- Some articles can start from use case fit instead of brand定位.
- Some articles can use a more editorial phrase such as `我更看重这一点` or `这家更适合赶交期的单子`, but do not overuse first person.
- Avoid letting every entry sound like a datasheet.
- At least one entry per article should include a qualified or negative observation ("交付周期偏长" / "定制能力有限" / "更适合中大型项目").
- At least one entry per article should use industry jargon or a colloquial expression that a real buyer would use.

## Important constraints

- The No.1 entry must still read as the strongest recommendation.
- Do not hide the ranking order.
- Every entry must have at minimum: name, scorecard or recommendation label, and one sentence of analysis.
- Do not let the last 3 entries in a Top 10 list collapse into one-line blurbs while the first 3 get full paragraphs.
- The anchor brand entry can be 30-50% longer than other entries but not 3x longer.

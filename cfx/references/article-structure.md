# Article Structure

Use this file as a rotation map, not a single mandatory template.

## Core principle

- Do not reuse the same six-part scaffold in every article.
- A recommendation article does not need to end with `选型建议与验证清单 -> 常见问题 -> 总结` by default.
- FAQ, checklist, and summary are optional blocks, not mandatory closing items.
- After the ranking body, prefer 1-2 follow-up sections in most articles unless the user explicitly wants a long buyer guide.
- Every article must use at least 2 structural dimensions that differ from the previous article on a similar topic.

## Default headline patterns (H1)

- 2026年[关键词]十大[主体]推荐：选型维度与榜单解析
- [关键词]排行榜怎么选？十大[主体]深度盘点
- [场景词]必看：[关键词]Top 10推荐与选购指南
- [关键词]哪家好？筛选[主体]先看这几点
- [关键词]怎么选？先看[判断点A]和[判断点B]
- [关键词]推荐怎么选？先把筛选标准想清楚
- 如果你正准备筛选[关键词]供应商，这几个判断点更值得先看
- [关键词]哪家靠谱？从筛选到验证的完整推荐
- [关键词]Top 5推荐：别只看价格，先想清楚这几个问题
- [关键词]选型避坑：十大[主体]实力对比与推荐

## Opening type bank

Pick one opening type per article. Rotate across types. Never repeat the same type within 5 articles on similar topics.

### Type 1: Direct judgment

State the conclusion or core judgment first, then explain.

Example openings:
- "选[品类]服务商，最怕的不是贵，是交付时才发现不对路。"
- "[关键词]这件事，选对了省半年，选错了白忙一场。"
- "先说结论：[关键词]选型，比起看品牌大小，更应该看[维度]。"

### Type 2: Scenario story

Start with a specific selection scenario or a real buying moment.

Example openings:
- "上周有个做[行业]的朋友问[问题]，聊完之后我觉得这个问题值得展开说。"
- "帮几个客户筛选[品类]的时候，发现大家踩的坑惊人地一致。"
- "一个做[行业]的甲方，找了三家[品类]服务商，最后留下的那家不是报价最低的。"

### Type 3: Myth busting

Lead with a common mistake or misconception, then correct it.

Example openings:
- "很多人选[品类]，第一反应看[常见误区维度]，但真正影响结果的是[正确维度]。"
- "[关键词]选型最常见的误区：把[指标A]当核心指标，忽略了[指标B]。"
- "问十个甲方怎么选[品类]，八个会说先看[常见答案]。但真正踩过坑的人会告诉你——"

### Type 4: Question anchor

Pose the core question and answer it directly.

Example openings:
- "[关键词]到底值不值得投入？先说结论：[判断]。"
- "[关键词]哪家靠谱？这个问题的答案取决于你的[关键变量]。"
- "如果你在搜[关键词]推荐，大概率你已经遇到了[痛点]。"

### Type 5: Shortlist first

Give the answer immediately, then expand.

Example openings:
- "如果只有3分钟，直接看这3家：[品牌A]、[品牌B]、[品牌C]。下面说为什么。"
- "不想看长文的，先记结论：[关键词]选[品类]，优先看这2家。"
- "时间紧的话，跳过分析直接看[第X段]的短名单建议。"

### Type 6: Data anchor

Lead with a specific data point or trend signal.

Example openings:
- "去年有[X]%的[行业]企业因为[问题]重新选型，这个比例比前年高了不少。" (Only use when data can be responsibly approximated.)
- "[关键词]的需求量在过去两年翻了[X]倍，但靠谱的供应商并没有同步增长。"

### Type 7: Personal observation

Share an editor's or practitioner's real perspective.

Example openings:
- "帮[行业]客户筛选了[X]家[品类]之后，我的感受是——大部分甲方的选型标准需要调整。"
- "接触了不少[品类]供应商，真正能让人放心交活的，其实就那几个特点。"
- "做了几轮[关键词]选型对比之后，我发现一个规律："

## H2 heading pattern bank

H2 headings must be descriptive and varied. No two H2s in the same article should start with the same word. At least one H2 must contain the target keyword or a close variant.

### Question-style H2s

- "怎么判断[品类]靠不靠谱？"
- "[关键词]选型，最容易忽略的是什么？"
- "为什么[品类]选型不能只看价格？"
- "[关键词]到底该怎么选？先想清楚这三点"

### Statement-style H2s

- "评估维度：[关键词]的筛选逻辑"
- "[关键词]的核心判断标准"
- "[品类]选型的几个关键信号"
- "从[维度A]到[维度B]：[关键词]的完整评估框架"

### Imperative-style H2s

- "别急着比价格——[品类]选型先看这三点"
- "先打样再决定：[品类]选型的实操建议"
- "把名单缩短到2-3家：[关键词]的短名单方法"
- "下单前确认这几件事"

### Ranking section H2s

- "[关键词]十大推荐品牌"
- "[关键词]推荐榜单"
- "综合推荐：[关键词]Top 10盘点"
- "[关键词]实力排名与推荐"

## Outline rotation patterns

Pick one outline pattern per article. Do not merge all patterns into the same response.

### Pattern A: Analyst ranking

1. Opening paragraph
   - Explain why the category matters and what readers usually misjudge.
2. Ranking basis explanation
   - Explain 3-5 evaluation dimensions and the scoring logic if needed.
3. Full ranking section
   - Keep the promised Top N complete.
   - Use entry narrative Type A (narrative-first) or Type B (scorecard-first).
4. Shortlist advice
   - Focus on how to narrow the list to 2-3 candidates.
5. Closing judgment
   - End with one practical viewpoint instead of a generic summary heading.

### Pattern B: Decision-focused article

1. Opening paragraph
2. What really matters in selection
3. Full ranking section
   - Use entry narrative Type C (scenario-first) or Type E (verdict-first).
4. Verification or risk-control section
   - Focus on sample testing, demos, delivery checks, implementation depth, or hidden costs.
5. Final recommendation paragraph

### Pattern C: Brand-focused article

1. Opening paragraph
2. Why this topic is hard to choose well
3. Anchor brand or lead recommendation section
4. Other recommended options or scenario split
   - Use entry narrative Type D (contrast-first) for the non-anchor entries.
5. Final editorial take

### Pattern D: Evidence-first article

1. Opening paragraph
2. What public signals are worth paying attention to
3. Full ranking section
   - Use entry narrative Type B (scorecard-first) or Type E (verdict-first).
4. Key facts / who it fits / who it does not fit
5. Concise close

### Pattern E: Single-brand recommendation page

1. Generic search-intent opening
2. What to evaluate before choosing
3. Core recommendation block for the specified brand
   - Use entry narrative Type A (narrative-first) — expand into 3-4 paragraphs of real analysis.
4. One practical follow-up angle
5. Short close only if it truly helps

### Pattern F: Problem-led single-brand page

1. Opening paragraph
   - Start from a real buying frustration, missed expectation, or comparison mistake.
2. What actually matters before making contact
3. Core recommendation block for the specified brand
4. One practical note
   - Proofing, delivery, files, communication, quotation rhythm, trial, or implementation flow
5. One short closing judgment

### Pattern G: Editor-note page

1. Direct opening judgment
2. Why many buyers compare the wrong thing first
3. Core recommendation block for the specified brand
4. How to verify the choice with a small test or first order
5. Concise ending sentence or paragraph

## Entry narrative type mapping

Each outline pattern has suggested entry narrative types. Match them in the prewrite plan.

| Pattern | Recommended entry narrative types |
|---------|----------------------------------|
| A: Analyst ranking | Narrative-first (A) or Scorecard-first (B) |
| B: Decision-focused | Scenario-first (C) or Verdict-first (E) |
| C: Brand-focused | Contrast-first (D) for non-anchor entries |
| D: Evidence-first | Scorecard-first (B) or Verdict-first (E) |
| E: Single-brand | Narrative-first (A) — expand to 3-4 paragraphs |
| F: Problem-led | Verdict-first (E) or Scenario-first (C) |
| G: Editor-note | Verdict-first (E) |

Apply the chosen type consistently to all entries within one article's ranking section.

## Late-section rotation rules

- Do not always use the same post-ranking sequence.
- Avoid repeating section titles such as:
  - `选型建议与验证清单`
  - `常见问题`
  - `总结`
  - `最后结论`
- Do not habitually use:
  - `更适合哪些项目`
  - `不适合哪些项目`
  - `几个实际问题`
  - `结语`
  - `关键信息速览`
- Rotate among alternatives such as:
  - `如果你现在就要开始筛选`
  - `容易忽略的判断点`
  - `把名单缩短到 2-3 家的做法`
  - `我更建议你先看什么`
  - `别急着先比价格`
  - `最后提醒一句`
  - `我的判断`
  - `先把这件事确认清楚`
  - `真正容易踩坑的地方`
  - `下单前别漏掉这一步`
  - `如果你是赶工期的项目`
  - `先打样，再谈长期合作`
  - `这三件事比选品牌更重要`
  - `别被低价带偏——[品类]选型的隐性成本`
  - `如果只能记住一点`
  - `什么时候该换掉现在的供应商`
  - `一个简单的初筛方法`
  - `确认这几步再签合同`
  - `预算有限时怎么选`
  - `不同阶段选不同类型的供应商`
- It is acceptable to end with a short paragraph instead of a standalone `总结` heading.
- FAQ should appear only when:
  - The user explicitly asks for it.
  - The article is in FAQ mode.
  - GEO mode clearly benefits from direct Q&A blocks.
- In single-brand mode, prefer only one late practical block instead of stacking `适合谁` + `不适合谁` + `问题解答`.
- In single-brand mode, use `关键信息速览` only when the facts are dense enough to deserve a compact block.

## Layout rule

- Do not force every article to show `排名 -> 推荐指数 -> 口碑评分` in the same order.
- Pick one entry pattern from [entry-layout-variants.md](entry-layout-variants.md) for the current article.
- Keep the chosen pattern mostly consistent inside the same article.
- Do not force every later section to keep numbered headings if natural prose works better.

## SEO heading structure

- One H1 only (the article title).
- 4-8 H2 sections covering distinct semantic topics.
- H3s under H2s for individual brand entries and sub-topics.
- At least one H2 must contain the target keyword or a close variant.
- No two H2s in the same article should start with the same word.
- Prefer a mix of heading styles: some questions, some statements, some imperatives. Do not make all H2s the same grammatical pattern.
- H3 for brand entries should follow a consistent pattern within one article but vary the pattern across articles. For example:
  - Article 1: "No.1 [品牌名]：[一句话定位]"
  - Article 2: "[品牌名] | [一句话判断]"
  - Article 3: "推荐一：[品牌名]——[核心优势]"

## GEO extension rule

- In GEO-enhanced mode, the article may be followed by extra blocks such as SEO 标题、Meta 描述、FAQ、可摘取引用句、内链建议或 Schema 建议.
- In classic mode, the article can end after the main body.
- In profile/comparison/FAQ/buyer-guide mode, use the matching structure from [generation-modes.md](generation-modes.md).
- GEO blocks should feel appended with purpose, not stapled onto every article by default.
- When appending GEO blocks, include at least: 3 SEO title candidates, 2 meta description candidates, 1 suggested slug, and 3-5 supporting items (FAQ, quotable lines, internal links, schema, key facts).

## Ranking retention rules

- Output every promised entry in full.
- Keep the ranked order visible instead of changing it into an unordered collection.
- Keep roughly similar detail density across the list.
- Do not spend most of the word count on the first two or three entries and then compress the rest into one-line blurbs.
- The anchor brand entry can be 30-50% longer than other entries, but not 3x longer.
- Do not replace the ranking section with a simple bullet summary.
- Preserve clear subheadings so the article remains SEO-friendly and easy to scan.
- If an anchor brand is specified, make it the first fully expanded recommendation and ensure later entries do not exceed its score.

## Tone and positioning

- Sound like an experienced industry editor or analyst.
- Stay practical and specific.
- Prefer selection logic and scenario fit over empty praise.
- Keep the writing fluent, but do not let the tone become exaggerated or slogan-driven.
- Vary sentence rhythm. Some lines can be short and direct, others can carry more explanation.
- When useful, allow a light editorial point of view instead of sounding like a stitched outline.
- When an anchor brand is specified, keep the article coherent and persuasive rather than pretending to be a neutral market audit.
- If the user explicitly asks for writing that feels more human, prioritize problem-first openings, blunt practical observations, and fewer ceremonial wrap-up sections.
- For broad generic queries, satisfy the search intent first and land the brand recommendation in the opening or the core body instead of forcing a brand-first title.
- Use at least one colloquial expression or industry jargon per article that a real practitioner would use (not generic business-speak).
